74HC595芯片：

SRCLK和RCLK都是上沿触发，即从0到1的时候触发
SRCLK是把SER(只有一位啊！！是0或者1)送到右边待命
因为下一次再把SER送过去就是把右边待命的挤掉到右边(更高位)
所以取SER的顺序是【从高位取到低位】，先取高位，它肯定在输出端的高位！
而且啊 每次SRCLK之后就要都要重新把SRCLK【归零】
当把所有的数据都送到输出口的时候，就要用到RCLK从0到1的上升沿把输出端滞留的数据【一次性】输出
记得要把RCLK【归零】噢！！

示范代码(函数包装)：

void Hc595Sendbyte(u8 dat)
{
u8 a;
SRCLK=0;
RCLK=0;
for(a=0;a<8;a++)
{
SER=dat>>7;
dat<<=1;
SRCLK=1;//Move the SER into The shu ju cun chu qi
nop();
nop();
SRCLK=0;//上升沿触发，位移缓存器
}
RCLK=1;
nop();
nop();
RCLK=0;//上升沿触发，用于输出
}

中断总结：（打包、打包后放哪、怎么触发、重新赋值锁时间、静态变量定义、if之中初始化归零）

  • 1.配置打包成函数，注意两种中断配置的不同（一个是三个，一个是六个），定时计数器多了TMOD 和高低位初值的定义，其他三项定义都关于寄存器相互对应；
  • 2.打包好的函数一定要放到main函数【里面】，而且在while函数【外面】！（毕竟这本质上就是一种赋初值，做一次就好了）
  • 3.外部中断有两种触发方式，又IT0和IT1决定；定时计数器的触发方式则是定时器溢出！（这时候TF0或TF1会变成1作为标志，向CPU发出中断请求，当CPU回应时自动归零）
  • 4.在定时器的中断函数中，一定要马上给TH0和TL0赋值，因为这时候TH0和TL0已经溢出，值就变成了0，如果不重新赋值，那么之后进入中断函数的间隔就是65536us了；
  • 5.static u16 i;定义静态变量非常炫酷，只有第一次进入中断函数的时候定义，之后都不重新定义了；
  • 6.每次进入中断函数的时候都i++,当加到自己想要的次数的时候做if语句，如果想要【闪烁】首先要想到的就是要【将i初始化】，否则只能进入一次if；
  • 7.多BB一句，两个东西要重新赋值，别忘了！！忘了第一个，时间就不对;忘了第二个，就做不了多次；
